# 磁力金牛财务报警助手开发与使用文档

**文档日期**: 2026年1月17日  
**版本**: v2.0 (最终版)

---

## 一、 开发流程记录

本项目旨在为磁力金牛广告投放平台开发一个自动化的财务监控脚本，解决平台数据不实时刷新以及缺乏主动报警机制的痛点。以下是今天的开发迭代全过程：

### 1. 需求分析与原型验证
- **痛点识别**：平台页面数据是静态的（Server-side rendering 或非实时 Ajax），需要手动刷新才能看到最新消耗；缺乏消耗飙升的实时通知。
- **技术选型**：使用 Tampermonkey（油猴）脚本注入 JavaScript，配合 jQuery 进行 DOM 解析，利用 `GM_xmlhttpRequest` 进行跨域请求推送。
- **核心逻辑**：定时抓取“今日总花费” -> 计算增量 -> 触发报警。

### 2. 功能迭代
- **阶段一：基础数据抓取**
  - 确定了目标数据的 XPath/CSS 选择器。
  - 实现了对日期（YYYY-MM-DD）和金额格式（去逗号转浮点）的解析。
- **阶段二：PushPlus 微信推送集成**
  - 对接 PushPlus API，实现了将文本警告发送到用户微信。
  - 迭代了邮件通道支持（Channel: `mail`, Option: `163`），实现了 HTML 格式的富文本邮件通知。
- **阶段三：自动刷新与状态保持**
  - 引入 `GM_setValue` / `GM_getValue` 实现跨页面刷新的数据持久化（存储 Token、历史消耗数据、配置项）。
  - 增加了自动刷新页面（`location.reload`）的机制，确保获取实时数据。
- **阶段四：UI 交互重构**
  - 从硬编码配置升级为 **可视化 GUI 面板**。
  - 采用了 GPT 风格的白色简约 UI，支持折叠、状态指示和实时倒计时。
  - 实现了所有参数（Token、刷新间隔、查询间隔、报警阈值）的用户自定义配置。
- **阶段五：报警逻辑深度优化**
  - **动态滑动窗口**：放弃了死板的“每小时”统计，改为“本次消耗”对比“X分钟前（或历史最早）消耗”。
  - **单次判定机制**：优化性能，页面刷新后仅进行一次核心判定，后续仅更新 UI 计时，避免重复报警。
  - **冷却机制**：内置 10 分钟报警冷却，防止频繁打扰。

## 二、 脚本使用方法

### 1. 界面介绍
脚本加载后，页面右上角会出现一个白色的悬浮面板：
- **今日消耗总额**：当前抓取到的实时消耗。
- **X分增长**：根据你设定的查询间隔，计算出的消耗增量。
- **自动刷新计时**：距离下一次强制刷新页面的倒计时。

### 2. 配置参数说明
在面板展开状态下，你可以设置以下参数（修改后需点击 **“保存配置”**）：

| 参数项 | 说明 | 推荐值 |
| :--- | :--- | :--- |
| **PushPlus Token** | 你的 PushPlus 用户令牌（用于发微信/邮件）。 | 必填 |
| **刷新间隔 (分)** | 脚本自动刷新网页的频率。刷新越快，数据越实时。 | 3-5 分钟 |
| **查询间隔 (分)** | 计算增长量的回溯时间。例如设为 60，通过当前消耗减去 60 分钟前的消耗来判断涨幅。 | 30-60 分钟 |
| **报警阈值 (元)** | 在上述查询间隔内，如果消耗增量超过此金额，即触发报警。 | 根据预算设定 |

### 3. 操作按钮
- **保存配置**：将输入框中的设置保存到本地（油猴缓存）。
- **手动刷新页面**：立即重载页面，并重置自动刷新计时器。会自动触发一次数据判定。
- **测试邮件**：发送一条测试消息，验证 Token 是否正确以及报警通道是否通畅。
- **重置历史**：清空所有历史消耗数据。**建议每天开播前点击一次**，或在调整重要配置后点击。

## 三、 脚本部署流程

### 步骤 1：准备环境
1.  **浏览器**：推荐使用 Chrome 或 Edge 浏览器。
2.  **插件安装**：在扩展商店搜索并安装 **Tampermonkey (篡改猴)** 插件。

### 步骤 2：安装脚本
1.  点击浏览器右上角的 Tampermonkey 图标 -> **添加新脚本**。
2.  删除编辑器中所有默认代码。
3.  将提供的 `磁力金牛财务报警助手.user.js` 代码完整复制粘贴进去。
4.  按下 `Ctrl + S` 保存。

### 步骤 3：获取 PushPlus Token
1.  访问 [PushPlus 官网](http://www.pushplus.plus/) 并登录。
2.  获得你的 **Token**。
3.  **重要**：如果不使用微信推送而想用**邮件推送**，请在 PushPlus 个人中心绑定你的邮箱（如 `1844337648@qq.com`）。

### 步骤 4：初始化配置
1.  打开磁力金牛后台的 **财务/推广管理** 页面。
2.  等待 3 秒，面板会自动出现在右上角。
3.  填入 Token，设置合适的阈值（建议先设低一点测试），点击保存。
4.  点击“测试邮件”，确认收到通知。

## 四、 总结 & 项目亮点

本项目成功将一个被动的静态数据页面，转化为了一套**主动式、智能化的财务风控系统**。

**关键亮点：**
1.  **所见即所得**：抛弃了通过代码改参数的繁琐方式，所有逻辑均可在漂亮的 UI 面板上通过鼠标完成。
2.  **高可靠性**：双重报警机制（浏览器本地+邮件远程），结合数据持久化，即使浏览器崩溃重新打开，也能基于历史数据继续监控。
3.  **精准防抖**：采用“刷新后单次判定”+“邮件冷却”策略，在保证监控实时性（通过高频刷新实现）的同时，彻底解决了误报和轰炸骚扰的问题。
4.  **灵活适用**：无论是需要每 5 分钟盯盘的激进策略，还是 1 小时看一眼的稳健策略，均可通过自定义“查询间隔”实现。

此工具将极大幅度降低投手盯盘的精力损耗，是保障广告账户资金安全的重要辅助工具。
