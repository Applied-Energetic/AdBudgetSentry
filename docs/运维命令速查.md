# AdBudgetSentry 运维命令速查

这份文档用于后续日常运维，默认你的项目路径是：

```text
E:\Code\AdBudgetSentry
```

默认后端地址：

```text
http://127.0.0.1:8787
```

默认公网地址：

```text
https://admin.seanvault.tech
```

## 1. 本机后台

### 1.1 启动后端

第一次安装依赖并启动：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "E:\Code\AdBudgetSentry\scripts\start-analysis-gateway-background.ps1" -EnsureDeps
```

普通启动：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "E:\Code\AdBudgetSentry\scripts\start-analysis-gateway-background.ps1"
```

强制重启：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "E:\Code\AdBudgetSentry\scripts\start-analysis-gateway-background.ps1" -ForceRestart
```

### 1.2 前台直接运行后端

适合开发调试：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "E:\Code\AdBudgetSentry\scripts\run-analysis-gateway.ps1" -EnsureDeps
```

## 2. Cloudflare Tunnel

### 2.1 后台启动 Tunnel

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "E:\Code\AdBudgetSentry\scripts\start-cloudflared-background.ps1" -CloudflaredExe "C:\Cloudflared\cloudflared.exe"
```

强制重启 Tunnel：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "E:\Code\AdBudgetSentry\scripts\start-cloudflared-background.ps1" -CloudflaredExe "C:\Cloudflared\cloudflared.exe" -ForceRestart
```

### 2.2 前台直接运行 Tunnel

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "E:\Code\AdBudgetSentry\scripts\run-cloudflared.ps1" -CloudflaredExe "C:\Cloudflared\cloudflared.exe"
```

说明：

- 如果你已经使用 Cloudflare 官方 Windows Service，就不要再重复后台启动 Tunnel 脚本
- Tunnel 和官方 service 二选一管理即可

## 3. 日志查询

### 3.1 查看后端标准输出

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "E:\Code\AdBudgetSentry\scripts\get-server-logs.ps1" -Component gateway-stdout -Tail 100
```

### 3.2 查看后端错误日志

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "E:\Code\AdBudgetSentry\scripts\get-server-logs.ps1" -Component gateway-stderr -Tail 100
```

### 3.3 查看 Tunnel 日志

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "E:\Code\AdBudgetSentry\scripts\get-server-logs.ps1" -Component cloudflared -Tail 100
```

### 3.4 关键字过滤

查 Python 报错：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "E:\Code\AdBudgetSentry\scripts\get-server-logs.ps1" -Component gateway-stderr -Pattern "Traceback"
```

查 Tunnel 错误：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "E:\Code\AdBudgetSentry\scripts\get-server-logs.ps1" -Component cloudflared -Pattern "error"
```

## 5. Windows 计划任务

### 5.1 安装后端开机自启

管理员 PowerShell：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "E:\Code\AdBudgetSentry\scripts\install-analysis-gateway-task.ps1" -EnsureDeps
```

### 5.2 安装 Tunnel 开机自启

管理员 PowerShell：


```powershell

schtasks /Query /TN "AdBudgetSentry-Cloudflared"
```

## 7. 常见问题

### 7.1 端口 8787 被占用

表现：

- 日志里出现 `error while attempting to bind on address ('127.0.0.1', 8787)`

处理：

- 说明已经有一个后端实例在运行
- 不要重复启动
- 如需重启，用：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "E:\Code\AdBudgetSentry\scripts\start-analysis-gateway-background.ps1" -ForceRestart
```

### 7.2 公网打不开后台

先依次检查：

1. 本机 `http://127.0.0.1:8787/admin` 是否正常
2. `cloudflared` 是否在线
3. Public Hostname 是否指向 `http://127.0.0.1:8787`
4. `admin.seanvault.tech` 是否已经在 Cloudflare Tunnel 中绑定

### 7.3 博客和后台冲突

建议：

- 博客继续使用 `seanvault.tech` / `www.seanvault.tech`
- 后台只使用 `admin.seanvault.tech`

不要把根域名直接给后台。

## 8. 当前推荐的管理方式

目前建议你按这套方式管理：

- 后端：使用本仓库的后台启动脚本和计划任务
- Tunnel：如果已经安装了 Cloudflare 官方 Windows Service，就继续用官方 service
- 日志：统一从 `logs/` 查看

如果后面你决定完全切换到自己脚本管理 Tunnel，再停掉 Cloudflare 官方 service，不要两个同时跑。
