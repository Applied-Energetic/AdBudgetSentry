# Copilot / Gemini 3 Pro 前端改造提示词

下面这份提示词是给你在 Copilot Chat 或 Gemini 3 Pro 里直接使用的，目标是让它围绕当前项目做前端代码改造，而不是生成一套脱离项目上下文的通用页面。

## 1. 总提示词

```text
你正在改造一个真实项目的前端，不是从零生成演示页。

项目背景：
- 这是一个本地运行的监控系统，名字是 AdBudgetSentry
- 前端包括两个部分：
  1. Tampermonkey 油猴脚本面板
  2. FastAPI 后端返回的后台管理页面
- 油猴脚本负责采集磁力金牛财务页面的当前总消耗，并计算当前值与窗口基线的差值
- 后端负责持久化、规则分析、大模型分析和告警回执展示

当前技术约束：
- 油猴脚本使用原生 JS + jQuery
- 后台管理页当前是 Python 直接生成 HTML
- 不要引入大型前端框架
- 保持 Windows 本地可运行
- 优先小步重构，不要大拆大改

本次前端改造目标：
- 提高信息密度和可读性
- 让状态、异常、告警、分析结果更容易扫读
- 保留现有业务逻辑和接口
- 不要删除已存在的上报逻辑、心跳逻辑、分析逻辑、PushPlus 逻辑
- 不要虚构不存在的后端接口
- 如果需要新增接口，先明确写出新增接口说明，再改代码

视觉要求：
- 风格要专业、克制、偏监控台
- 避免 AI 演示风、避免花哨卡片堆砌
- 配色以青绿、深蓝、灰白为主
- 强调告警态、健康态、最近分析和最近告警
- 保持桌面优先，同时兼容窄屏

输出要求：
- 先分析现有页面问题
- 再给出改造方案
- 再输出可直接落地的代码修改
- 不要只给概念稿
- 如果涉及多个文件，说明每个文件改什么
```

## 2. 后台管理页改造提示词

```text
请改造 AdBudgetSentry 后台管理页，目标文件是 Python 生成 HTML 的页面。

当前页面已经有这些区域：
- 总览统计
- 实例健康列表
- 最近告警记录
- 调试接口说明

请基于真实监控后台做优化，要求：
- 提高总览层级，让用户一进来先看到“实例健康、最近告警、最近采集”
- 把“最近告警记录”做成更容易扫读的卡片或列表
- 把“实例健康列表”做成更适合运维查看的表格
- 强化账号信息、实例 ID、异常类型、最近分析摘要的排版
- 做好颜色层级，但不要过度艳丽
- 保持纯 HTML + CSS，不引入前端框架
- 不改动接口数据结构，优先只改展示层

请输出：
1. 页面结构调整建议
2. CSS 视觉方案
3. 最终可直接替换的代码
```

## 3. 油猴面板改造提示词

```text
请改造一个油猴脚本面板 UI，现有技术栈是原生 JS + jQuery + GM_addStyle。

当前面板已有功能：
- 显示今日总消耗
- 显示 10 分钟增量
- 保存配置
- 手动刷新
- 测试报警
- 重置历史
- 立即分析
- 拖拽面板
- 分析结果文本框

改造目标：
- 保留所有已有功能和按钮
- 保留拖拽和位置记忆
- 不改变已有接口调用
- 让“今日总消耗 / 10 分钟增量 / 最近状态 / 分析结果”更清晰
- 分析结果区域更适合阅读长文本
- 配置区更紧凑，减少视觉噪音
- 不要做成花哨弹窗风格
- 要像一个真实监控工具的小面板

请输出：
1. 现有 UI 问题分析
2. 结构重排方案
3. 可直接替换的 HTML 片段
4. 可直接替换的 CSS 片段
5. 若需要，补充少量 JS 交互优化
```

## 4. 趋势图改造提示词

```text
请为 AdBudgetSentry 后台页设计一个“最近采样趋势”模块。

要求：
- 用最轻量的方式实现
- 优先纯 HTML / CSS / 少量内联 JS
- 如果需要图表库，优先选择轻量方案，并说明接入成本
- 图表重点展示：
  - 当前总消耗趋势
  - 窗口增量趋势
  - 告警触发时间点
- 风格与现有后台保持一致
- 不要为了图表重写整页

输出要求：
1. 模块放置位置建议
2. 数据结构建议
3. 最小可行实现方案
4. 代码示例
```

## 5. 实例详情页提示词

```text
请基于现有监控后台，设计一个“实例详情页”或“实例侧边详情面板”。

目标：
- 点击实例后，查看该实例的完整状态
- 展示账号名称、账号 ID、实例 ID、页面类型、最近心跳、最近采集、最近错误、最近分析、最近告警
- 重点是排障效率，不是营销展示
- 保持风格专业、克制、偏运维后台

约束：
- 先基于当前已有字段设计
- 不要凭空依赖未实现的字段
- 如果要新增字段或接口，先列清楚

请输出：
1. 信息架构
2. 交互方案
3. 布局方案
4. 对现有代码的改造建议
```

## 6. 使用建议

推荐你实际使用时按下面顺序喂给 Copilot / Gemini：

1. 先发送“总提示词”
2. 再发送“后台管理页改造提示词”或“油猴面板改造提示词”
3. 最后把目标文件内容一并贴进去，让它在真实上下文里改

如果你要它少走偏，最好再加一句：

```text
请不要输出全新项目结构，只在现有文件基础上做增量修改。
```
